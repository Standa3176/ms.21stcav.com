## Langfuse runtime DB-override LIMITATION (documented)

**Status:** v1 ships with Langfuse credentials sourced from `.env` ONLY at boot via the `mliviu79/laravel-langfuse-prism` plugin's auto-discovery service provider.

**Why:** The Langfuse Prism middleware reads `config/agents.php` at boot and binds its tracer client there. Mutating `config()` at runtime to override with a DB-row credential fights the package's auto-discovery contract — calls already in flight would still use the boot-time config, producing inconsistent tracer state.

**Operational impact:**
- Langfuse credentials in the `integration_credentials` row are READ-ONLY for v1: surfaced in the admin UI for visibility + change history + the per-row Test connection action (which works via direct HTTP `GET {host}/api/public/health` — bypassing the Prism middleware).
- The production tracer continues to read from `LANGFUSE_PUBLIC_KEY` / `LANGFUSE_SECRET_KEY` / `LANGFUSE_HOST` env vars — operator must `git push` an env update + bounce workers to actually rotate Langfuse credentials.
- The other 4 integrations (Supplier API / Woo REST / Bitrix webhook / Anthropic API key) DO honour DB-row override at runtime within ≤60s (resolver cache TTL + observer invalidation).

**v2 enhancement path:** Ship a service provider boot hook that reads the `integration_credentials` row for `langfuse_observability` (if active) and mutates `config('agents.langfuse.*')` BEFORE the mliviu79 service provider boots. Requires ordering control on Laravel's deferred provider boot — feasible but not v1 scope.

## APP_KEY rotation runbook

Rotating `APP_KEY` invalidates ALL encrypted payloads on the `integration_credentials` table (and the Phase 11.2 `competitor_ftp_credentials` table). Post-rotation runbook:

1. Generate new APP_KEY: `php artisan key:generate --show` (review output before applying)
2. Apply: `APP_KEY=base64:... php artisan config:clear`
3. Bounce queue workers: `php artisan horizon:terminate`
4. Open `/admin/integration-credentials` and re-enter EVERY credential payload (5 rows max)
5. Click "Test connection" on each row — confirm green
6. Open `/admin/competitor-ftp-credentials` and re-enter every credential there too
7. Operator confirms `dashboard:refresh` + check IntegrationHealthWidget shows 5 green tiles before proceeding

Documented in: `app/Domain/Integrations/Models/IntegrationCredential.php` class docblock (APP_KEY ROTATION RUNBOOK section).

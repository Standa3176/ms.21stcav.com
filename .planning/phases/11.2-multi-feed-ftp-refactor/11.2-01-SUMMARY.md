# Phase 11.2 Plan 01: Multi-Feed FTP Refactor Summary

## One-liner

Replaces Phase 11.1's per-source FTP model with the (shared credential) + (many feeds) shape matching real-world operator UX — `competitor_ftp_credentials` (~1 row, encrypted at rest) + `competitor_ftp_feeds` (one integer-PK row per remote file matching the screenshot's IDs) + `FeedFormatNormaliser` (csv/tsv/zip/txt → canonical CSV before Phase 5 ever sees the file). Phase 5 ingest pipeline byte-identical untouched per D-07.

## What shipped

- **3 migrations**: 1 destructive drop of Phase 11.1 schema + 2 create (ULID-PK credentials + integer-PK feeds with UNIQUE local_filename + SoftDeletes)
- **2 models**: CompetitorFtpCredential (HasUlids + 3 encrypted casts + LogsActivity allow-list) + CompetitorFtpFeed (int auto-increment + SoftDeletes + BelongsTo competitor & credential)
- **2 hand-written policies**: admin-only credentials (D-09); admin write + pricing_manager view-only feeds (D-11)
- **2 factories**: with `nearAutoDisable()` + `stale()` states
- **1 new service + exception class**: FeedFormatNormaliser handles 4 formats (csv passthrough / tsv → csv / zip extract first *.csv / txt delimiter sniff @ 80% confidence) reusing Phase 5 EncodingDetector for tsv encoding
- **3 refactored services**: FtpSourceConnector::connect signature (CompetitorFtpSource → CompetitorFtpCredential); CompetitorFtpPullCommand (--source flag dropped; --feed + --credential flags + mutual exclusion + D-13 9-step pull algorithm); CompetitorFtpPullFailedNotification (source → feed)
- **2 new Filament Resources** matching the operator screenshot: CompetitorFtpFeedResource (7 columns + default sort id ASC + red stale text + Refresh now action + Download bulk action) + CompetitorFtpCredentialResource admin-only with Test connection action
- **10 file deletions**: Phase 11.1 source model + policy + factory + Resource + 4 superseded tests
- **7 Pest test files** (51 tests, 129 assertions): 2 model tests + FeedFormatNormaliser + command refactor + 2 Resource RBAC + Phase5IngestUntouchedTest architectural test (D-07)
- **PolicyTemplateIntegrityTest floor bump 30 → 31** + Gate::policy pair swap
- **RolePermissionSeeder updates**: read_only revoke + pricing_manager whereIn whitelist for `view_*_competitor_ftp_feed` perms
- **config/competitor.php**: +`ftp.stale_days` (default 30)

**1. [Rule 3 - Blocking] Replaced AlertRecipient::factory() with explicit ::create() in command refactor test**

- **Fix:** Switched to `AlertRecipient::create(['email' => ..., 'name' => ..., 'is_active' => true, 'receives_competitor_ftp_alerts' => true])` matching the existing `AgentNotificationTest.php` pattern.
- **Files modified:** `tests/Feature/Competitor/Ftp/CompetitorFtpPullCommandRefactorTest.php`
- **Commit:** 4a73ae6

**2. [Rule 3 - Blocking] Removed step-4d pre-syncPermissions givePermissionTo for pricing_manager**
- **Found during:** Task 3 RolePermissionSeeder update
- **Issue:** Initial implementation called `$pricingManager->givePermissionTo([...])` between the read_only revoke (step 4c) and the pricingManager `syncPermissions` block (step 5). `syncPermissions` REPLACES all attached perms, so the pre-sync grants would be wiped.
- **Fix:** Moved the 4 view perms into the `pricingManagerPermissions` whereIn whitelist directly, anchored on the existing Phase 11.1 absence comment block. Idempotent — `syncPermissions` picks up the literals from the same array assembled in step 5.
- **Files modified:** `database/seeders/RolePermissionSeeder.php`

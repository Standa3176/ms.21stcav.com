## Plan vs reality

The original plan called for a migration to make `competitor_prices.product_id` nullable. Reading the actual schema revealed `competitor_prices` has no `product_id` column at all — match status is computed at query time via `LOWER(TRIM(competitor_prices.sku)) = LOWER(TRIM(products.sku))`. So the fix was just dropping an early-return in the writer; no schema change needed.

## Changes (3 files, +73/-14)

### 1. CompetitorCsvRowWriter.php — orphan branch persists

```diff
- if ($product === null) {
-     $this->orphanDetector->record(...);
-     $run->increment('rows_orphaned');
-     return;
- }
+ $isMatched = $product !== null;
+ if (! $isMatched) {
+     $this->orphanDetector->record(...);
+     $run->increment('rows_orphaned');
+ }
+ // ... fall through to CompetitorPrice::create() either way ...
+ if ($isMatched) {

+ }
```

Side-effect semantics preserved:
- `rows_orphaned` still increments per orphan (audit metric)
- `OrphanDetector::record()` still fires (suggestion creation)
- `CompetitorPriceRecorded` event still suppressed for orphans (margin analyser needs Product)
- `rows_written` now counts all persisted rows regardless of match status

- New TextColumn 'Match' badge: success="Matched" if matched, warning="Orphan" if not

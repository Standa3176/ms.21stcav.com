# Quick Task 260504-d7v

## Goal

Closes the "products table is empty" gap by adding a bulk Woo catalogue import. After this runs once, the `Match Status` filter on Catalogue → Competitor Prices starts showing real Matched/Orphan ratios.

## Tasks

### Task 1 — `app/Domain/Sync/Commands/WooImportProductsCommand.php`

Signature:
```
woo:import-products
    {--with-supplier : Also enrich buy_price + supplier_stock from supplier feed}
    {--limit=0 : Stop after N simple products (0 = no limit)}
    {--dry-run : Don't persist; report what would change}
```

Behaviour:
1. Inject `WooClient`, `SupplierClient`. WooClient auto-fails on missing creds via existing IntegrationCredentialResolver.
2. If `--with-supplier`: call `$supplier->fetchAllProducts()` upfront. Returns SKU-keyed map. Print "Loaded N supplier SKUs".
3. Page Woo via `$this->woo->get('products', ['per_page' => 100, 'page' => $page])` — start at page 1, stop on empty. Use the FULL Woo response objects (NOT WooProductIterator which strips fields).
4. For each product:
   - If `type === 'variation'` or type is grouped/external → skip (variations: $skipped_variation++, log; grouped/external: $skipped_other++)
   - Else (`simple` or `variable` parent): build upsert payload from Woo response:
     - `sku` (string), `name`, `type`, `status` (publish/draft/private), `stock_status`, `slug`, `short_description`, `long_description`
     - `sell_price` from `regular_price` or `price`
     - `last_synced_at` = now()
   - If `--with-supplier` and SKU exists in supplier map → also set `buy_price` from supplier price
   - If `--dry-run`: $would_create/$would_update++ based on `Product::where('woo_product_id', $id)->exists()`; no DB write
   - Else: `Product::updateOrCreate(['woo_product_id' => $id], $payload)` inside try/catch — count $created vs $updated by checking `wasRecentlyCreated`
5. After each page: print "Page N: imported X / total Y so far"
6. Honour `--limit`: stop processing simple products after N (variations don't count)
7. Final summary: created/updated/skipped_variation/skipped_other/errored
8. Errors: log + ImportIssue::create({sku, issue_type=TYPE_UNKNOWN_SKU, error_message=...}) — match SyncSupplierCommand's pattern (verify ImportIssue's TYPE constants, use the closest fit, e.g. TYPE_PARSE_ERROR if it exists)

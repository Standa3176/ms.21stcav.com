## Tasks

### Task 1 — SQLite WAL default

`config/database.php` line 41: `'journal_mode' => null` → `'journal_mode' => 'WAL'`. Inline comment explaining why (writer-vs-writer lock contention from job_batches updates during parallel CompetitorCsvChunkJob batches).

### Task 2 — `competitor:retry-quarantine` command

`app/Domain/Competitor/Console/Commands/CompetitorRetryQuarantineCommand.php`

Extends `BaseCommand` → implements `perform(): int`.

Signature:
```
competitor:retry-quarantine
    {--all : Retry every quarantined file}
    {--file= : Retry a specific filename}
```

Behaviour:
1. Default (no flags): list files at `storage/app/competitors/quarantine/<date>/*.csv` + report last error from each `.error.json` sidecar. No mutation. Print "Use --all to retry every file, or --file=NAME to retry one."
2. `--all`: move every .csv from quarantine subdirs back to `storage/app/competitors/incoming/`. Skip if target already exists in incoming/ (idempotent — print warning). Delete corresponding `.error.json` sidecar after successful move. Print count moved.
3. `--file=screenmoove_2026-01-01.csv`: same as --all but only matching that filename across all date subdirs.
4. Error if both --all and --file given.
5. Always print at end: "Now run: php artisan competitor:watch (after 30s mtime gate)."

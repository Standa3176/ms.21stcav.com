### Task C — Add navigationBadges to 12 resources

Standard badge pattern (Filament 3.3):

```php
public static function getNavigationBadge(): ?string
{
    $count = SomeModel::query()->where(...)->count();
    return $count > 0 ? (string) $count : null;
}

public static function getNavigationBadgeColor(): ?string
{
    return 'warning'; // or 'danger' / 'success' / 'gray' / null
}
```

Mapping (use the precise condition for each):

1. **SuggestionResource** — count where `status='pending'` (verify field name) — color `warning`
2. **CompetitorFtpFeedResource** — count where `last_pull_status='failed'` — color `danger`
3. **CsvParseErrorResource** — count where `resolved_at IS NULL` — color `danger`
4. **AutoCreateReviewResource** — count where `status='pending'` (verify) — color `warning`
5. **AgentRunResource** — count where `status='failed'` (verify) — color `danger`
6. **SyncRunResource** — count where `status='failed' AND created_at >= now()->subDay()` — color `danger`
7. **CompetitorIngestRunResource** — count where `status='failed' AND started_at >= now()->subDay()` — color `danger`
8. **ImportIssueResource** — count where `resolved_at IS NULL` (verify col) — color `warning`
9. **CompetitorResource** — total count (informational) — color `gray`, hide if 0
10. **CompetitorPriceResource** — total count formatted with thousands sep — color `gray`, hide if 0
11. **ProductResource** — total count formatted with thousands sep — color `gray`, hide if 0
12. **IntegrationCredentialResource** — count where `is_active=false` — color `warning`, hide if 0

For thousands-sep on count badges (Products, Competitor Prices), use `number_format($count)`.

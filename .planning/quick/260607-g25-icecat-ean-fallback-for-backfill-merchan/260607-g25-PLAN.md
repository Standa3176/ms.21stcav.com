<objective>
Add an Icecat GTIN fallback to `products:backfill-merchant-feed --field=ean` so the
top 137 golden-ad-target SKUs (Panasonic PT-, Sony FW-Bravia, PTZOptics, BirdDog,
Roland VR-, Yealink MD/MTOUCH, etc.) whose supplier_db.feeds_products.ean is `"N/A"`
or empty can still be auto-EAN-populated by querying Icecat with brand+MPN.

Purpose: Today's prod run returned 0/116 EAN updates against this premium-B2B
cohort because distributors (Midwich / Tech Data / Nuvias / Northamber / WestCoast)
don't publish GTIN for channel-only products. Icecat indexes EAN against MPN for
the major AV brands — at ~0.2p/query it's ~99.5% cheaper than Claude and unlocks
the Google Merchant Center recovery path for the highest-value SKUs.

Output:
- New public `IcecatClient::lookupGtinByMpn(?string $brand, ?string $mpn): ?string`
- `BackfillMerchantFeedCommand` extended with `--icecat-fallback` (default ON),
  `--no-icecat-fallback`, `--max-icecat-spend-pence=200`, 3 new outcome buckets,
  cost banner, and a 7-row outcome table when fallback fires.
- 6 new Pest feature cases + 1 unit test, mocking Icecat at the container.
- 4 atomic commits (one per code task) + 1 verification task.
</objective>

# Icecat response shape (from `fetchImageUrls` request path):
#   GET https://live.icecat.biz/api?lang=EN&shopname=...&app_key=...&Brand=...&ProductCode=...
#   JSON body has: { "data": { "Image": {...}, "Gallery": [...], "GeneralInfo": {...}, "EANCodes": [...] } }

#   - data.GeneralInfo.GTIN          (string, e.g. "5025232931842")

#   - data.EANCodes[]                (array of EAN strings — older schema; we accept first non-empty)
# `lookupGtinByMpn` MUST tolerate ALL THREE shapes (Icecat has not deprecated any of them; the
#  exact field present varies per product). The executor MUST cover all three in the
#  unit test fixtures (Test 1 / Test 2 / Test 3 below).

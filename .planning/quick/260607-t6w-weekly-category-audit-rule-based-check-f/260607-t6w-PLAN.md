<objective>
Weekly rule-based category audit for the ecom manager. New artisan command
`products:audit-categories` scans every `status='publish'` Product, classifies
it via 4-bucket most-severe-wins rule (missing | orphaned | uncategorized |
suspicious brand-category mismatch), TRUNCATE-and-replaces a new
`category_audit_findings` table, then surfaces the snapshot on a new Filament
page at `/admin/category-audit` plus a home-dashboard tile. Scheduled Fri
22:00 London so Mon morning the ecom manager has a fresh report to triage.

**Free, deterministic, runs in <30s** — no Claude spend at scan time. Claude
is only invoked per-SKU when the operator clicks the row action to re-run
`products:assign-taxonomy` on a misclassified product.

Purpose: Bad category mapping breaks faceted nav, breaks Merchant Center
category mapping, and causes Google Shopping disapprovals. Today the ecom
manager has no internal signal — they catch issues by browsing /wp-admin.
This closes the loop inside the Filament chrome.

Output: 1 migration + 1 model + 1 command + 1 page + 1 widget + 1 schedule
entry + 1 blade view + supporting Pest tests. 7 atomic commits + 1
verification-only task.
</objective>

  - private function classify(Product $p, array $byId, array $brandNameById, array $homes): ?array
    Returns null when HEALTHY, otherwise:
      ['issue_type' => 'missing'|'orphaned'|'uncategorized'|'suspicious',
       'severity'   => 1|2|3|4,
       'category_name' => string,
       'category_root_name' => ?string]

    Most-severe-wins ordering (return on FIRST match):
      a) $p->category_id === null
         → ['issue_type'=>'missing', 'severity'=>1, 'category_name'=>'']
      b) !isset($byId[$p->category_id])
         → ['issue_type'=>'orphaned', 'severity'=>2, 'category_name'=>'(unknown #'.$p->category_id.')']
      c) preg_match('/^uncategori[sz]ed$/i', $byId[$p->category_id]['name']) === 1
         → ['issue_type'=>'uncategorized', 'severity'=>3, 'category_name'=>$byId[$p->category_id]['name']]
      d) Suspicious bucket — requires $p->brand_id non-null AND brand name (case-insensitive) IS A KEY in $homes:
         - $brandName = $brandNameById[$p->brand_id] ?? null
         - $brandHomes = caseInsensitive lookup into $homes by $brandName
         - If $brandHomes !== null:
           - $rootName = $this->rootName($p->category_id, $byId);  // walks parent up to 0 with defensive cycle guard (max 20 hops)
           - If $rootName NOT IN $brandHomes:
             → ['issue_type'=>'suspicious', 'severity'=>4, 'category_name'=>$byId[$p->category_id]['name'], 'category_root_name'=>$rootName]
      e) Otherwise HEALTHY → return null

  - private function rootName(int $categoryId, array $byId): ?string
    Walks $byId[$categoryId]['parent'] up to 0. Defensive cycle guard via $seen[] tracking + hard cap at 20 hops (TaxonomyResolver's buildPath pattern). Returns the topmost ancestor's name, or null if the chain breaks.

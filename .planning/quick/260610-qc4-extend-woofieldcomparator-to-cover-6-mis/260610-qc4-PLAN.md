# 260610-qc4 — Extend WooFieldComparator to cover 6 missed fields

## Context

**The discovery:** The 2026-05-30 cutover's shadow-mode divergence scan compared only 7 fields
(`name`, `slug`, `short_description`, `long_description`, `meta_description`, `sell_price`, `image_url`)
out of the 13 fields Woo and MS share as source-of-truth state. The 6 missed fields each caused
a customer-first incident **AFTER** cutover instead of being caught pre-cutover:

| Field | Customer-first symptom | Discovery quick |
|-------|------------------------|-----------------|
| `stock_quantity` | Ergotron HA310-2EP "in stock 7" phantom-stock case | 260609-nku audit + 260609-rie root fix |
| `stock_status` | `instock` + qty=0 + backorders=yes limbo class | 260609-nku |
| `buy_price` | Margin math could silently drift | (latent — no incident yet) |
| `category_id` | 3,244 / 3,922 live products NULL | 260607-v5g backfill |
| `brand_id` | 82.4% of products NULL — brand link missing on storefront | 260607-cgd brand backfill |
| `ean` | 89% Google Merchant Center disapproval rate | 260607-cgd EAN backfill |

This task closes the comparator so any future divergence-scan run reports the truth on all 13 fields.

**Contract per field (the defensive rules — read carefully, this is the whole point):**

| Field | Woo path | Local source | When to EMIT a diff | When to SKIP silently |
|-------|----------|--------------|---------------------|------------------------|
| `stock_quantity` | top-level `stock_quantity` | `$local->stock_quantity` (int cast) | Both sides resolvable AND values differ (cast both to int; null = "not present" not 0). EMIT when local has value but Woo top-level key absent — top-level absence on a canonical Woo column IS a divergence (the 260609-nku Ergotron case). | Both null. |
| `stock_status` | top-level `stock_status` | `$local->stock_status` (string) | Both sides resolvable AND `(string) !==` after lowercase. EMIT when local has value but Woo top-level key absent. | Both null/empty. |
| `buy_price` | `meta_data[key=BUY_PRICE_META_KEY].value` (cast float) | `$local->buy_price` (decimal:4 → float) | Both resolvable AND `abs(local - live) > 0.005` (mirror sell_price tolerance). | Meta key ABSENT on Woo side (plugin not installed on this Woo) — silent skip per defensive contract. Both null. |
| `category_id` | `$wooProduct['categories'][0]['id']` (int) | `$local->category_id` (int) | Both resolvable AND int values differ. EMIT when local has value but Woo `categories` key absent or `[0]` missing — Woo categories is canonical Woo state and absence IS a divergence. | Both null. |
| `brand_id` | `meta_data[key=BRAND_ID_META_KEY].value` (int) via helper `extractBrandId()` | `$local->brand_id` (int) | Both resolvable AND int values differ. | Meta key ABSENT on Woo side (Woo may carry brand via pa_brand attribute only — deferred to a follow-up quick). Silent skip. Both null. |
| `ean` | First non-empty value across `EAN_META_KEYS` via helper `extractEan()` | `$local->ean` (string) | Both resolvable AND `(string) !==` after trim. | All 3 EAN meta keys ABSENT on Woo side (Woo has not been backfilled yet for this SKU). Silent skip. Both null/empty. |

**Why silent-skip the meta-only fields when Woo lacks the key:** Most Woo installs don't have
the WC COG plugin or the EAN plugins. Emitting a `buy_price`-divergence diff for every product
just because Woo lacks `_alg_wc_cog_cost` would flood `sync_diffs` with thousands of false
positives on the next live scan and bury the real divergences (which is exactly why the original
comparator was scope-limited — but the original over-corrected and dropped the 6 fields entirely).
The silent-skip + canonical-emit split is the correct middle ground.

**Why canonical-Woo fields (stock_quantity, stock_status, categories) DO emit when local has
a value but Woo doesn't:** Those are top-level Woo columns. Absence is meaningful — it means
the Woo product exists but is in a degenerate state (no stock managed, no category assigned).
That IS a divergence the cutover scan needs to see.

### D. Two PRIVATE helper methods at the BOTTOM of the class

```
private function extractBrandId(array $wooProduct): ?int
{
    foreach ($wooProduct['meta_data'] ?? [] as $meta) {
        $key = $meta['key'] ?? null;
        if ($key === self::BRAND_ID_META_KEY) {
            $val = $meta['value'] ?? null;
            return ($val === null || $val === '' || !is_numeric($val)) ? null : (int) $val;
        }
    }
    return null;
}

private function extractEan(array $wooProduct): ?string
{
    foreach (self::EAN_META_KEYS as $eanKey) {
        foreach ($wooProduct['meta_data'] ?? [] as $meta) {
            if (($meta['key'] ?? null) === $eanKey) {
                $val = trim((string) ($meta['value'] ?? ''));
                if ($val !== '') {
                    return $val;
                }
            }
        }
    }
    return null;
}
```

Both helpers assume the stdClass→assoc normalisation already ran in the diff() prelude. Both
return null on absence so the per-field comparison blocks above can use the same `$live === null`
silent-skip check uniformly.

<objective>
Event-driven MS→Woo propagation for `stock_quantity` / `buy_price` / `sell_price` / `category_id`. Eloquent observer on `Product` dispatches a domain event when one of these fields changes; a queued listener calls a shared `WooProductWriter` service to PUT the change to Woo. Feature-flagged (default OFF) for safe deploy.

Purpose: Today the only MS→Woo propagation paths are the 23:00 `cutover:auto-sync` (260611-rl4) and the 07:20 hydrate (260611-qcq). A direct MS-side product edit (admin save, agent action, ad-hoc tinker) does not reach Woo until the next 23:00 run. This task adds instant propagation (seconds, not hours) for the 4 deterministically-pushable fields the divergence pipeline already understands, while preserving the daily auto-sync as backstop.

Output:
- `WooProductWriter` service (single source of truth for the 4-field PUT payload).
- `ProductFieldsChangedEvent` domain event + `ProductObserver`.
- `PushProductFieldsToWoo` queued listener.
- Feature flag `cutover.event_driven_push_enabled` (default false).
- Bulk-path guards (`Product::withoutEvents`) on `WooImportProductsCommand`.
- 10 Pest cases (4 observer + 6 listener), 5-6 atomic commits.
</objective>

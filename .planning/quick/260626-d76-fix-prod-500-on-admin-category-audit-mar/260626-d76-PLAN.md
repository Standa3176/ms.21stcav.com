<objective>
Fix the production 500 on `/admin/category-audit`.

ROOT CAUSE (confirmed from prod laravel.log, DB `stcav_meeting`):
`CategoryAuditPage::buildQuery()` (app/Filament/Pages/CategoryAuditPage.php:215-225) scopes the
table to the latest audit run with:

```php
->whereIn('run_id', function ($q): void {
    $q->from('category_audit_findings')
        ->select('run_id')
        ->orderByDesc('audited_at')
        ->limit(1);
});
```

MariaDB (prod) rejects `IN (subquery + LIMIT)`:
`SQLSTATE[42000]: 1235 This version of MariaDB doesn't yet support 'LIMIT & IN/ALL/ANY/SOME subquery'`.
Filament's pagination count query (`getCountForPagination`) trips it on every render → 500.

The test suite never caught this because the test DB is SQLite (`:memory:`), which *permits*
`IN (subquery + LIMIT)`. So Pest is green while prod is broken.

FIX: resolve the latest `run_id` as a scalar in a separate query, then filter by equality —
no `IN`-with-`LIMIT`. Guard the empty-table case so an empty table still renders zero rows.

Add a SQL-shape regression guard (assert the compiled SQL has no `in (select ... limit`) so this
MariaDB-only incompatibility is catchable on SQLite going forward, plus a behavioural test that
multi-run data shows only the latest run.

This is the ONLY occurrence of the anti-pattern in app/Filament (AdCandidatesPage uses
`whereIn` with a plain array, which is fine).
</objective>

TARGET buildQuery() shape (MariaDB-safe):
```php
private function buildQuery(): Builder
{
    // The latest run_id resolved as a SCALAR first — MariaDB rejects
    // IN(subquery + LIMIT) (SQLSTATE 42000, error 1235). SQLite allowed the
    // old subquery form, which is why this 500'd only on prod. Snapshot
    // semantics (AuditProductCategoriesCommand TRUNCATEs + re-inserts one run)
    // mean a single run_id dominates in practice; the equality predicate is
    // the contract and guards against accidental historical rows.
    $latestRunId = CategoryAuditFinding::query()
        ->orderByDesc('audited_at')
        ->value('run_id');

    return CategoryAuditFinding::query()
        ->with('product:id,name')
        ->when(
            $latestRunId !== null,
            fn (Builder $q): Builder => $q->where('run_id', $latestRunId),
            fn (Builder $q): Builder => $q->whereRaw('1 = 0'),
        );
}
```
(`Illuminate\Database\Eloquent\Builder` is already imported at the top of the file.)

# Phase 260702-h50 Plan 01: Foundation for "Brands to Add" (extract trait + refresh command) Summary

Extracted the feed-manufacturer → Woo-brand resolution into a shared `ResolvesWooBrandKey` trait and built `products:refresh-brands-to-add` — the Piece-1 precompute that resolves each pending `new_product_opportunity` SKU's brand, tags the suggestion (`evidence.brand` + `evidence.brand_on_woo`), and caches a brands-to-add summary so the Piece-2 admin page renders instantly. No Claude spend, no storefront writes.

## Why brand needs a precompute

The brand of a `new_product_opportunity` suggestion is **not stored on the suggestion** — it is derived at runtime from the supplier feed manufacturer (via `supplier_products`). So Piece 2's "filter/group Suggestions by brand" and "which brands would unlock the most pending products if I added them to Woo" both need the brand resolved **ahead of time**. This command does that resolution once (nightly Mon-Fri), writing the answer onto each suggestion's `evidence` and into a single cache entry.

## Task 1 — `ResolvesWooBrandKey` trait (no behaviour change)

`resolveBrandKey(string $mfrLower, array $wooBrandsByLower): ?string` (260628-b9t: exact match, then strip a trailing `" - <suffix>"` and retry) and `firstResolvableBrandKey(array $manufacturers, array $wooBrandsByLower): array{0:?string,1:?string}` (260629-rct: pick the first manufacturer that resolves to a Woo brand — prefers a real brand over a warranty/add-on row sharing an MPN) were **moved verbatim** (kept public) into `App\Domain\ProductAutoCreate\Concerns\ResolvesWooBrandKey`. `DraftFromSuggestionsCommand` now `use`s the trait and the inline copies were deleted.

## Task 2 — `products:refresh-brands-to-add` command + schedule

`RefreshBrandsToAddCommand extends BaseCommand`, injects `IntegrationCredentialResolver` + `TaxonomyResolver`, and `use`s `ResolvesWooBrandKey`. Signature: `products:refresh-brands-to-add {--dry-run} {--limit=0}`.

`perform()`:
1. Load `$wooBrandsByLower` from `TaxonomyResolver::allBrands()` (abort if empty).
2. Walk pending `new_product_opportunity` suggestions → `[suggestionId => sku]` (reads `evidence.sku`; `--limit` caps the walk).
3. Batch `supplier_products` manufacturer lookup (`fetchManufacturers` — mirrors `DraftFromSuggestionsCommand`'s mysqli walk; every walked SKU gets an entry, default `[]`, so not-in-feed SKUs classify as not-sourceable).
4. `$index = buildBrandsToAddIndex($skuToManufacturers, $wooBrandsByLower)`.
5. If **not** `--dry-run`: merge `brand` + `brand_on_woo` into each suggestion's `evidence` and `Cache::put('suggestions.brands_to_add', {generated_at, brands:[...]}, 86400)`.
6. Print the to-add summary table (brand | pending-SKU count) **always**.

### `buildBrandsToAddIndex` rules (the PURE, unit-tested core)

Per SKU (lowercased key):

| Case | `sourceable` | `on_woo` | `brand` | In `to_add`? |
|------|-------------|----------|---------|--------------|
| No supplier manufacturer (`[]`) | `false` | `false` | `null` | No — a **not-sourceable** bucket (different problem) |
| Manufacturer resolves to a Woo brand | `true` | `true` | canonical Woo name | No — already on Woo |
| Has manufacturer(s), none resolve | `true` | `false` | the **first** manufacturer (what the operator would add) | **Yes** — `count++`, push sku |

`to_add[brand] = {count:int, skus:[...]}`; sample `skus` capped at **25** (the `count` is the true total). Unit test asserts the plan's 5-SKU case (`bh71` on-woo, `ab12`+`cd34` → Trantec count 2, `ef56` multi→Yealink on-woo, `gh78` not-sourceable) plus a 40-SKU cap case and an all-on-woo/not-sourceable empty-`to_add` case.

### Cache shape (the Piece-2 contract)

```
Cache key: 'suggestions.brands_to_add'  (RefreshBrandsToAddCommand::CACHE_KEY, TTL 86400s)
{
  "generated_at": "2026-07-02T06:50:00+00:00",
  "brands": [                       // ksorted by brand name
    { "brand": "Trantec", "count": 2, "skus": ["ab12", "cd34"] },
    ...
  ]
}
```

Piece 2 reads this for the "Brands to Add" page and uses `evidence.brand` / `evidence.brand_on_woo` for the Suggestions Brand / on-Woo filters.

NEW: app/Console/Commands/PublishDraftsCommand.php (extends BaseCommand — same base as RetryMissingImages /
HydrateProductStockFromOffers). Mirror DraftFromSuggestionsCommand's --auto-approve publish loop
(lines ~388-426) for the dispatch + counters + shadow handling.

```php
protected $signature = 'products:publish-drafts
    {--status=draft : Comma-separated auto_create_status values eligible to publish (default: draft).}
    {--skus= : Comma-separated explicit SKU list (still filtered to complete + not-pushed).}
    {--require-images : Only publish drafts that already have >=1 gallery image (default: publish regardless — Woo uses its placeholder).}
    {--limit=0 : Max products this run (0 = unbounded).}
    {--dry-run : List matching SKUs + counts; dispatch nothing.}';

protected $description = 'Bulk-publish complete auto-created drafts (brand+category set, never pushed) to Woo via PublishProductJob.';
```

perform():
1. Parse: $statuses = comma-split of --status (trim, drop blanks; default ['draft']); $explicit = parse --skus;
   $requireImages = (bool) --require-images; $limit = max(0,(int)--limit); $dryRun = (bool) --dry-run.
2. Capture the triggering user id for PublishProductJob's publishedByUserId (auth()->id() ?? 0) — BEFORE any
   dispatch (mirror RetryMissingImages' auth capture note).
3. Build query:
```php
$q = Product::query()
    ->autoCreated()                       // existing scope (auto_create_status != 'manual')
    ->whereNull('woo_product_id')         // never pushed
    ->whereNotNull('brand_id')            // complete taxonomy — never publish a needs-assignment row
    ->whereNotNull('category_id')
    ->whereIn('auto_create_status', $statuses);
if ($explicit !== []) { $q->whereIn('sku', $explicit); }
if ($requireImages) {
    // driver-aware, mirror AutoCreateHealthPage::emptyImagesExpr but the NON-empty case
    $expr = DB::connection()->getDriverName() === 'sqlite'
        ? 'json_array_length(gallery_image_urls) > 0'
        : 'JSON_LENGTH(gallery_image_urls) > 0';
    $q->whereNotNull('gallery_image_urls')->whereRaw($expr);
}
if ($limit > 0) { $q->limit($limit); }
```
4. Fetch (cursor()/get()). If empty → info 'No complete not-pushed drafts match.' return SUCCESS.
5. --dry-run: render a table (sku, auto_create_status, images=count(gallery_image_urls), brand_id, category_id)
   + "Would publish N product(s)." return SUCCESS. Dispatch nothing.
6. Live: loop, per product:
```php
try {
    PublishProductJob::dispatchSync((int) $product->id, $userId);
    $product->refresh();
    if ((int) ($product->woo_product_id ?? 0) > 0) { $published++; $this->line("  ✓ {$product->sku} → Woo #{$product->woo_product_id}"); }
    else { $shadowed++; $this->warn("  ⊘ {$product->sku}: no Woo id (shadow mode?) — not marked published"); }
} catch (\Throwable $e) { $failed++; $this->warn("  ✗ {$product->sku}: ".$e->getMessage()); }
```
   Then a counters summary (scanned, published, shadowed, failed).
NOTE: intentionally NO completeness-threshold gate — this is an explicit operator action on drafts they can
see. Document that in the class docblock.

# Phase 260707-f06 Plan 01: Fix always-blank Feeds count column Summary

Fixed the always-blank Feeds column on Settings > Competitor Feeds by renaming the column key `feeds_count` → `ftp_feeds_count` so it matches the `->counts('ftpFeeds')` / `withCount('ftpFeeds')` alias Laravel produces.

## What was done

- **Root cause:** `TextColumn::make('feeds_count')->counts('ftpFeeds')`. `->counts('ftpFeeds')` adds `withCount('ftpFeeds')`, which exposes the attribute `ftp_feeds_count` (Laravel snake-cases the `ftpFeeds` relation). The column was keyed `feeds_count`, so it read a non-existent attribute and rendered blank on every row — a pre-existing key/alias mismatch since the column was first written, unrelated to the 260705-pw3 / 260706-pfy feed-date work.
- **Fix (one key):** `make('feeds_count')` → `make('ftp_feeds_count')`. `->label('Feeds')`, `->counts('ftpFeeds')`, `->tooltip(...)`, and `->sortable()` preserved verbatim. The `withMax('ftpFeeds','remote_file_date')` query modifier, the Feed file date column, the Last Ingest colouring, other columns, actions, and badges are untouched (`withCount` and `withMax` coexist fine).
- **Guard test:** `tests/Feature/Competitor/CompetitorFeedsCountColumnTest.php` — seeds an active competitor with 2 `CompetitorFtpFeed` rows and one with 0, authenticates as an `admin` (mirrors the existing `CompetitorFtpFeedResourceTest` auth: seed `RolePermissionSeeder`, `actingAs` a role-assigned user), and asserts `assertTableColumnStateSet('ftp_feeds_count', 2 / 0, ...)` on `ListCompetitors`. The record is passed by primary key (not the raw model) so Filament resolves it through the table query and the `withCount` alias is populated — mirroring exactly what renders in the UI.

## Deviations from Plan

None functional. One judgement call: the test passes the record **by primary key** rather than the model instance, because Filament's `assertTableColumnStateSet` uses a passed Model directly (which has no `withCount` alias) but resolves a passed key through the table query pipeline (which applies `->counts('ftpFeeds')`). This is the only way the assertion reflects the actual rendered state and is what proves the fix.
